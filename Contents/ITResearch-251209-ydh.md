# Inbound와 Outbound

## 기본 정의

IT 분야에서 Inbound(인바운드)는 외부에서 내부 시스템(서버)으로 들어오는 네트워크 트래픽을 의미합니다.
<br/>
반대로 Outbound(아웃바운드)는 내부 시스템에서 외부로 나가는 네트워크 트래픽을 의미합니다.

## 구체적인 예시

### Inbound 트래픽의 예

- 클라이언트가 웹사이트(HTTP/HTTPS)에 접속하는 경우 (포트 80, 443)
- 외부 사용자가 SSH로 서버에 원격 접속하려 할 때 (포트 22)
- 애플리케이션에서 외부 데이터베이스에 접속하려 할 때 (MySQL 포트 3306, PostgreSQL 포트 5432)
- 파일 업로드 시 클라이언트가 서버로 데이터를 전송할 때

### Outbound 트래픽의 예

- 서버가 클라이언트에게 HTTP 응답을 보낼 때
- EC2 인스턴스가 외부 웹사이트에 API를 호출할 때
- 서버에서 파일을 다운로드할 때
- 소프트웨어 업데이트를 위해 인터넷 리포지토리에 접근할 때

## 방화벽(Firewall)에서의 역할

IT 인프라에서 가장 중요하게 사용되는 개념이 방화벽 설정입니다. 방화벽은 네트워크 트래픽을 허용하거나 차단하는 보안 시스템으로, 인바운드와 아웃바운드 규칙을 통해 보안을 관리합니다.

| 항목 | Inbound | Outbound |
|-----|---------|----------|
| **트래픽 방향** | 내부로 들어오는 것 | 외부로 나가는 것 |
| **기본 설정** | 모든 접속 차단 (필요한 것만 허용) | 모든 접속 허용 |
| **예시** | 클라이언트 → 서버 | 서버 → 클라이언트 |
| **Windows 방화벽** | 기본값: 차단 | 기본값: 허용 |

## IT 네트워크 관점의 인바운드 차단

- 보안 원칙: 필요한 것만 명시적으로 허용
- DDoS 공격 방지: 외부로부터의 대량 접속 차단
- 악성 트래픽 필터링: SQL 인젝션, XSS 등의 웹 공격 방지
